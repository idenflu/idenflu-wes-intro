NOVA Approach

작게 시작하고, 하나로 운영하고, 계속 확장합니다

NearGo식 단계 도입 메시지와 CJ올리브네트웍스식 전체 구축 관점을 NOVA의 플랫폼 구조로 정리합니다.

01
Start Small

작게 시작

기존 시스템과 설비를 기준으로 피킹, 분류, 출고, 설비 모니터링 등 특정 구간부터 적용합니다.

02
Operate One

03
Scale Modular

Architecture Boundary

업무, 설비, 실행 흐름을 섞지 않습니다

NOVA의 핵심은 기능 목록이 아니라 책임 경계입니다. 경계가 유지되어야 새 설비와 새 센터가 추가되어도 구조가 흔들리지 않습니다.

Platform

권한, 감사, 알림, 설정, 스케줄링, 실시간 구독처럼 여러 현장에서 반복되는 공통 기능을 소유합니다.

Operation

입고, 출고, 이동, 피킹, 분류 같은 업무 상태와 완료 기준을 소유합니다.

Agent

AGV, AMR, Conveyor, Sorter, WMS, ERP처럼 설비와 외부 시스템별 연결 방식을 흡수합니다.

Orchestration

여러 업무와 설비를 순서대로 연결하고 재시도, 타임아웃, 보상, 수동 개입 지점을 관리합니다.
