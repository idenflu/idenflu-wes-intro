Why It Breaks
운영 화면과 설비 연동이 분리돼 있습니다 사용자는 여러 화면을 보고, 실제 연동은 다른 시스템이 처리합니다.
공통 기능이 플랫폼이 아니라 프로젝트별 부속물입니다 인증, 권한, 알림, 감사, 스케줄링이 반복 구축됩니다.
새 설비 추가가 확장이 아니라 재개발이 됩니다 기존 운영 영향 범위를 다시 검토하고 일정이 길어집니다.
결국 운영팀은 시스템을 관리하는 대신 예외를 수습합니다 늦게 감지하고, 늦게 판단하고, 늦게 대응하는 구조가 고착됩니다.

증상이 반복되는 이유는 구조 안에 있다.

Architecture Principles
공통 기능 표준화 인증, 권한, 감사, 알림, 스케줄링, 실시간 스트림을 플랫폼 기본값으로 제공합니다.
운영과 연동의 분리 콘솔 경험과 설비 연동 계층을 분리해 변경 영향을 최소화합니다.
도메인 모듈 독립성 설비별 요구사항은 BC 단위로 수용하고 기존 운영은 흔들지 않습니다.
새 센터에 같은 구조를 반복 적용 한 번 검증한 플랫폼을 다른 설비와 다른 센터로 확장합니다.

공통은 표준화하고, 다름은 모듈로 분리한다.

Realtime Loop

실시간 운영은 보고가 아니라
이벤트 흐름으로 움직입니다

Snapshot-first Delta push 구조로 화면과 현장을 지속 동기화합니다. 장애는 감지되고, 알림은 전달되고, 재배분은 실행되고, 이력은 자동으로 남습니다.

01

설비 이벤트 수집

AGV·컨베이어·소터 상태를 실시간 스트림으로 수신합니다.

02

이상 징후 감지

임계치와 예외 조건을 기준으로 이상 상태를 즉시 판별합니다.

03

멀티채널 알림

심각도 기준으로 운영자와 관리자에게 자동 전파합니다.

04

작업 재배분

유휴 설비나 대체 라인으로 태스크를 신속히 분산합니다.

05

감사 이력 기록

누가 무엇을 조치했는지 자동으로 추적하고 내보냅니다.

Core Messages

NOVA의 가치는 세 가지로 압축됩니다

통합

모니터링, 작업, 알림, 보안, 감사, 스케줄링을 하나의 운영 구조로 묶습니다.

단일 콘솔에서 설비 상태와 태스크를 함께 제어
공통 플랫폼 계층으로 반복 구축 제거
단일 계약 구조로 운영 복잡도 축소

안정성

운영 판단의 기준을 보고서에서 이벤트 스트림으로 바꿉니다.

Snapshot-first Delta push 기반 동기화
WebSocket 실시간 스트림과 멀티채널 자동 알림
장애 감지부터 조치 이력까지 연속 추적

확장성

새 설비나 새 센터를 붙여도 플랫폼 전체를 다시 만들지 않습니다.

독립 설비 BC 구조
기존 운영 영향 최소화
수평 확장 가능한 모듈형 아키텍처

세 단어로 기억한다 — 통합, 안정성, 확장성.
